# X06 ROME Factual Editing

## 目标

把“解释”推进到“可控编辑”，并同时考虑 edit success、locality 与 collateral damage。

## 核心参考

- [Locating and Editing Factual Associations in GPT](https://arxiv.org/abs/2202.05262)

## 为什么它对扎实的 P6 成长重要

- 只会读图还不够，真正的研究还要能讨论编辑是否稳定、是否局部、是否值得上线。
- 这篇会把 interpretability 拉向 intervention 与 product-facing judgment。

## Notebook 与交付

- Notebook：`notebooks/extensions/zh/x06_rome_factual_editing.ipynb`
- 交付：1 份 rank-one factual edit 记录，外加 1 份 edit memo，讨论 success、locality 与副作用。

## 验收题

- 为什么“编辑成功”本身还不足以证明方法可靠？
- 你的 toy rank-one update 里，最明显的 collateral damage 出现在哪？
- 如果 locality 和 edit success 冲突，你会优先保哪一个？

## 扩展结论

这篇会让你开始习惯：研究结论不只是“能不能改”，而是“改完之后代价和边界是什么”。
