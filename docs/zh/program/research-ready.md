# Research-Ready Path

这个仓库现在不是单一课程，而是三层叠起来的训练系统：

- 一层是 Colab 优先的论文复现主线
- 一层是研究学徒式 bootcamp
- 一层是用来把 `P6` 地板打稳打深的扩展复现与作品集层

第二层和第三层之所以重要，是因为“我跟着 notebook 跑过了”和“我能在严肃研究场景里稳定产生价值”之间，缺的通常不是看过更多内容，而是复现密度、写作密度和研究判断。

## 这条路径到底想培养什么

这条路径要把一个普通本科生或个人研究者推进到这里描述的 `P6`：能承担明确研究任务、批判证据，并提出下一步实验。

这里最关键的边界是：12 周训练营只是核心学徒阶段，不是全部毕业线。只有当学习者额外通过完整的毕业清单时，仓库才应该把结果叫做 `P6-ready`。

在核心阶段，一个学习者应该能做到：

- 快速读一篇论文，并说清真正的问题、方法、证据、局限和下一步实验
- 干净地复现一个结果，并解释到底是哪一个改动改变了输出
- 在自己的写作里区分 observation、inference 和 speculation
- 写出导师、研究同伴和未来协作者都能快速扫完的短 memo
- 提出一个带 baseline、预算和 stop condition 的下一步实验

再往上走时，目标会变成：

- 把多篇论文串成一条研究语言，而不是各篇孤立理解
- 把 interpretability、editing、auditing 和 control 放进同一套判断框架
- 写出两周级研究计划，并知道什么证据足以推进或终止一个方向

## 进入前的要求

只有在你已经具备下面这些条件时，才建议进入高密度训练：

- 会用 Python、NumPy、PyTorch 做和调试小实验
- 能在本地或 Colab 跑 notebook
- 对矩阵、范数、logit、cross-entropy 有基本概念
- 每周能稳定投入 8-12 小时
- 愿意写实验记录和读论文笔记，而不是只看

如果这些条件还没到位，先补基础。这条路径是故意做得很密的。

## 两条起跑线

为了避免把所有读者都当成同一种起点，这个项目继续保留两条起跑线：

### 小白起跑线

- 适合还没进入阿里式 `P4` 状态的人。
- 如果你还不能稳定跑 notebook、读基础图表、解释 baseline，就从这里开始。
- 先完成 `F00-F03` 以及 `M00`、`M01`，把 reading note 和 experiment log 模板用熟。
- 第一目标不是立刻进入研究路径，而是先把自己推进到接近 `P4`，然后再进入主线复现。

### 基础者起跑线

- 适合已经能跑 notebook、会基本 PyTorch、能做简单复现的人。
- 如果你已经有一些实验和阅读能力，但缺一条系统 interpretability 主线，就从这里开始。
- 建议先更快完成可运行的 `M00 → M05` 主线，再把 `M03` 和 `M06-M10` 当成阅读与批判任务插入，然后尽早进入高密度写作模式。
- 目标不是停在“我把 notebook 跑通了”，而是把整条路径压实成一个可辩护的 `P6-ready` 结果。

## 现在这条路径下面压着四层支撑

- `Pre-P4 基础包`：先补环境、attention 形状感、向量几何和实验纪律。
- `主线论文讲义 + live 复现`：每篇 Anthropic 主线论文都有镜像讲义，只有诚实可跑的条目才配 Colab。
- `参考答案层`：用样例校准 memo、brief、experiment log 和 proposal 的密度。
- `扩展论文复现轨道`：把 transformer circuits、memory/editing 和 auditing 继续做成 Colab 复现。

## 从“课程模式”切到“研究模式”之后会发生什么

在课程模式里，理解一个想法就够了。

在研究模式里，每篇论文至少都要留下 4 个产物：

1. 一份 paper reading note
2. 一份 experiment log
3. 一份短 memo
4. 一份下一步问题清单

在扩展阶段，这个要求会继续升级成：

1. 一份可复核的 Colab 复现
2. 一份明确区分 observation / inference / speculation 的 memo
3. 一段“这次复现仍然不能证明什么”
4. 一个能接到下周计划里的下一步实验

如果你拿不出这些输出，你大概率还在浏览，而不是在训练。

## 不能妥协的工作规则

- 每次只改一个变量家族。
- 跑 variant 之前，先把 baseline 写清楚。
- 永远把 observation 和 interpretation 分开。
- 永远问一句：什么证据会推翻我最喜欢的解释？
- 永远写一段“这个 artifact / 结果 / edit 还不能证明什么”。
- 每周复盘，不要只在最后复盘。

这些规则之所以重要，是因为初学者最容易把图看得太满、把失败记得太少，而从 `P5` 到 `P6` 最显著的差别之一恰恰是证据边界意识更强。

## 结营之后怎么把 P6 打稳

12 周 bootcamp 结束后，不要停在“我完成了课程”。单独完成训练营还不等于 `P6-ready`。下一步应该立刻做三件事：

- 进入 `X01-X07` 扩展复现轨道
- 反复对照参考样例，提高 memo 和 proposal 的密度
- 把 capstone 从一次作业，升级成连续几轮小研究循环

如果主线再加毕业清单让你获得了 `P6-ready` 结果，那么这些动作才是在把这块地板打稳，而不是只留下浅层完成感。

## 结营时你应该拿到的作品集

完成这条完整路径之后，你应该拿到的是一组作品，而不是一段记忆：

- 10 份以上 paper reading notes
- 10 份以上 experiment logs
- 6 份以上短 research memos
- 2 份以上 graph / artifact / edit critique
- 2 个两周可执行的 capstone proposals
- 至少 3 次满足分组要求的扩展复现
- 1 份完整的 scoped research-task dossier

这份作品集，才是你给导师、协作者或同行评审展示“我不只是看过，我已经能按研究方式工作”的东西。

## 这条路径依然不能替代什么

没有任何仓库能字面替代真实导师、协作者群体或长期 apprenticeship。但这条路径真正能做的是移除最常见的阻塞项，让普通本科生有机会诚实达到 `P6-ready`：

- 论文读不明白
- 实验记不清楚
- 不会做 failure analysis
- memo 写不出来
- 不会把多篇论文连成一条方向
- 想不出下一步实验

如果你是认真完成这条路径的，这些问题就不该再成为别人第一眼看到的短板。
