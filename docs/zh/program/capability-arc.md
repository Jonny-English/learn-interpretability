# 能力成长弧线

## 先给直接结论

- 可以：一个普通本科生认真完成这套仓库之后，确实有可能达到这里描述的 `P6`。
- 但前提是：不能只算“看完了、跑完了”，而必须通过明确的证据门槛。
- 所以这个仓库的直接目标是把认真完成者推进到 `P6-ready` 或早期 `P6`，而不只是 `P5`。
- `P7` 和 `P8` 仍然保留为更长期的背景，不属于仓库直接完成后的宣称。

## 从 P4 往上的能力弧线

- `P4`：能在指导下跑通 notebook、复述概念、记录基础实验。
- `P5`：能独立完成小范围论文复现、记录清楚 baseline 与 variant，并写出基本结论。
- `P6`：能承担明确研究任务，批判证据，并提出下一步实验。
- `P7`：能把多篇论文串成一个方向，提出两周级研究计划，并管理证据边界和工具需求。
- `P8`：能多年负责一个 interpretability 子方向或工具线，对多名协作者形成稳定技术带动。

## 仓库里的训练段直接支持到哪里

- `F00-F03` 主要服务于 `Pre-P4 -> P4`。
- `M00-M10`、12 周训练营、rubric、参考输出和 capstone 流程，构成 `P6-ready` 路径里的核心学徒阶段。
- 至少 3 次扩展复现，再加 dossier、critique、memo 和 proposal 这些门槛，才是诚实成立 `P6-ready` 的最小收尾段。
- `X01-X07` 的完整轨道会继续加深这块 `P6` 地板，并帮助强完成者开始往早期 `P7` 走。
- 这里保留 `P8` 只是为了交代更远的能力背景。

## P6 证据门槛

只有当学习者能拿出下面这些证据时，才应该把结果叫做 `P6`：

- 完成 live 主线和阅读与批判层
- 至少 3 次扩展复现，覆盖 circuits、editing 或 memory、auditing 这几类
- 1 份 scoped reproduction dossier：里面有 baseline、1 个受限 sweep 或 ablation、failure-analysis 和下一步实验
- 1 份 artifact 或 graph critique：清楚区分 evidence、inference 和 speculation
- 1 份两页 memo 和 1 次短汇报：不是只总结内容，而是做出判断
- 1 份两周 proposal：范围够小、能真正失败、风险和 stop condition 都写清楚
- rubric 四个维度全部至少达到“就绪”，并且没有维度停留在“未就绪”

如果这四个门槛没有同时满足，那么结果更像 `P5` 或 `P5+`，不该直接叫 `P6`。

## 为什么普通本科生达到 P6 是合理的

- 课程把算力门槛压到了可以反复在 Colab 上练的程度。
- 训练营强制要求 notes、logs、memos、critique 和 proposal，而不是只跑 notebook。
- 仓库把可运行模块和只读批判模块结合起来，这比纯代码或纯阅读更接近真实的 scoped research task。
- 最终交付物的形状已经很接近早期研究任务：复现、判断、沟通、提出下一步。

## 仓库之外还有什么

- `P7+` 仍然需要更多原创实验、更多外部反馈和更强的问题选择能力。
- 一个仓库可以诚实地把人推进到 `P6-ready`，但不能单独认证更长期的高阶资历。
