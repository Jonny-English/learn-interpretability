# P8 路线图

## 先说清楚这不是哪种承诺

- 这个仓库不承诺“12 周结束就自动到 P8”。
- 更准确的说法是：它把学习路径设计成一条朝阿里式 `P8` interpretability 研究者长期推进的训练系统。
- 12 周主线训练营对应的是入场能力；P7/P8 需要更长周期的扩展复现、写作、提案和独立判断。

## P4 到 P8 的阶段推进

- `P4`：能在指导下跑通 notebook、复述概念、记录基础实验。
- `P5-P6`：能独立完成小范围论文复现、写清 baseline 与 failure analysis，并承担明确研究任务。
- `P7`：能把多篇论文串成一个方向，提出两周级研究计划，讨论证据边界和工具需求。
- `P8`：能负责一个 interpretability 子方向或工具线，对多名协作者形成稳定技术带动。

## 仓库里的对应训练段

- `F00-F03` 主要服务于 `Pre-P4 -> P4`。
- `M00-M10` 主要服务于 `P4 -> P5/P6`。
- `X01-X07`、参考答案层、company-onramp 和反复做 capstone，主要服务于 `P6 -> P7/P8`。

## 如果你的目标真的是 P8

- 不要只做主线；主线只是地基。
- 必须把扩展论文也做成 Colab 复现，而不是只看摘要。
- 必须累计作品集，而不是只累计完成数。
- 必须反复练习“写出下一步实验”，因为 P8 的核心不是会看图，而是会定义问题和推动方向。

## 你应该拿什么来判断自己有没有继续往上长

- 你是否开始把不同论文放进同一个研究语言里。
- 你是否能指出一个解释的证据边界，而不是只会复述作者结论。
- 你是否能提出一个团队真的可以执行的两周计划。
- 你是否能把 interpretability、editing、auditing 和 control 放进同一个判断框架里。
