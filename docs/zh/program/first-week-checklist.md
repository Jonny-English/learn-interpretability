# 第一周验证路径

如果你现在最想先回答的问题是：

这个仓库到底适不适合我？

那就先用这一份路径，不要一上来就给自己加完整项目压力。

这是一条给小白或本科生的“一周适配验证路径”。目标不是证明你已经达到 `P6-ready`，而是验证这个仓库能不能真的带你往那边走。

## 这一周做什么

1. 在 Colab 里跑 [F00](../../../notebooks/foundations/zh/f00_environment_plots_baselines.ipynb)。
2. 跑 [M00](../../../notebooks/zh/m00_zoom_in_circuits.ipynb)。
3. 跑 [M01](../../../notebooks/zh/m01_toy_models_superposition.ipynb)。
4. 写 1 份简短 reading note、1 份 experiment log、1 段 `100-300` 字 memo。

## 什么算成功

- 你能把 Colab 环境和 notebook 跑通，而不是全程都在迷路。
- 你能高层次解释什么是 feature、circuit、superposition。
- 你能分清 baseline、variant 和 conclusion。
- 你能写出 1 段短 memo，并明确区分 observation 和 inference。
- 你在做过真实工作之后，仍然愿意继续，而不是只是看 README 时觉得热血。

## 什么算警报信号

- 即使用 Colab，你还是始终无法稳定跑通 notebook。
- 你能复制结果，但说不清到底改了什么、为什么重要。
- 你没法用 100 字总结 M01 到底教会了你什么。
- 你持续回避 notes、logs、memos，只想看图不想写判断。

## 接下来怎么走

- 如果环境、图表、实验纪律还是不稳，就继续补 `F01-F03`，不要急着往后冲。
- 如果 `F00`、`M00`、`M01` 整体可控，就继续进入 live 主线：`M02`、`M04`、`M05`。
- 如果做到这里你已经觉得这个仓库确实有用、可信，而且给了你清晰路径，那就给它一个 star，然后继续走完整路径。

## 诚实的判断标准

第一周不要问“这个仓库是不是完美”。

先问它有没有已经给你三样东西：

- 一个真实可跑的 Colab 路径
- 一条比随机跳论文更清晰的学习顺序
- 一个更具体的 interpretability 工作图景

如果答案是有，那它就在完成自己的工作。
