# Reference-Output Layer

This layer does not exist so you can copy templates. It exists so you can calibrate what a deliverable looks like when it is ready to hand to a mentor, advisor, collaborator, or peer reviewer.

## What problem this layer solves

- Many beginners know they should write memos, but not how dense those memos need to be.
- When there is only a rubric and no example, the common failure mode is "I wrote a lot, but I still made no judgment."
- The point of a reference example is not to think for you. It is to show how evidence, risk, judgment, and next steps are arranged.

## Recommended use

- Write your own version first, then compare it to the reference. Do not start by reading the answer.
- When you inspect the examples, focus on structural density rather than copying wording.
- Every time you compare, ask: am I missing evidence, judgment, risk, or a next-step plan?

## When you most need it

- Before the first independent research sprint.
- Before the first serious memo submission.
- Whenever you feel you "wrote a lot" but another reader still cannot see your judgment.
