# Week-by-Week Bootcamp

Treat this as a 12-week apprenticeship, not a playlist. Each week ends with a gate. If the gate is not met, repeat the week or narrow the scope.

## Week 1: Environment, terms, reading method

- Modules: `M00`
- Goal: make the environment boring and stable so the actual learning can start
- Core work:
  - run the repository locally or in Colab
  - write your first paper reading note
  - build a glossary of terms you do not genuinely understand yet
- Outputs:
  - one reading note
  - one working notebook run
  - one list of five unclear terms
- Gate: explain feature, circuit, and intervention in your own words without reading from the notebook

## Week 2: Visual circuits to superposition

- Modules: `M00`, `M01`
- Goal: understand why clean neuron pictures stop being enough
- Core work:
  - reproduce the toy-model plots
  - sweep one variable family such as hidden dimension or sparsity
  - write down the geometric reason overlap appears
- Outputs:
  - one experiment log
  - one annotated plot
  - one short memo on why neuron semantics break
- Gate: you can explain superposition as geometry, not only as "neurons mix concepts"

## Week 3: Sparse autoencoders and monosemanticity

- Modules: `M02`
- Goal: see why feature recovery can be cleaner than raw neuron analysis
- Core work:
  - run the SAE toy lab
  - sweep one factor such as sparsity, feature count, or noise
  - record where recovery quality degrades
- Outputs:
  - one experiment log
  - one note on failure boundaries
  - one short memo on what the recovered features buy you
- Gate: you can name at least one concrete reason monosemanticity gets worse in your sweep

## Week 4: From examples to feature maps

- Modules: `M03`
- Goal: stop thinking in isolated feature anecdotes and start thinking in catalogs
- Core work:
  - browse the teaching feature catalog
  - build your own grouping scheme
  - propose one missing feature worth looking for
- Outputs:
  - one taxonomy note
  - one missing-feature question
  - one short memo
- Gate: you can explain why a catalog changes the research style, not only the scale

## Week 5: Features as probes

- Modules: `M04`
- Goal: turn features into readout tools
- Core work:
  - run the probe notebook
  - change the target label or inject one confounder
  - write down what accuracy means and what it does not mean
- Outputs:
  - one probe memo
  - one feature-weight plot
  - one list of three confounders
- Gate: you can discuss classifier performance together with a baseline and confounders

## Week 6: Features as interventions

- Modules: `M05`
- Goal: reason about utility instead of only gain
- Core work:
  - run a steering-strength sweep
  - define one off-target metric
  - identify the sweet spot and explain why it appears
- Outputs:
  - one steering memo
  - one off-target metric
  - one failure-analysis note
- Gate: your conclusion includes both benefit and collateral cost

## Week 7: Midpoint review

- Modules: `M01`, `M02`, `M04`, `M05`
- Goal: consolidate the first half into evidence instead of loose memories
- Core work:
  - collect the notes and logs from Weeks 1-6
  - score yourself against the rubric
  - run one gap-closing experiment on the weakest dimension
- Outputs:
  - one two-page midpoint memo
  - one rubric self-score
  - one corrective action
- Gate: you can name your weakest research habit and show one concrete fix

## Week 8: Reading tracing graphs

- Modules: `M06`
- Goal: learn how to read a graph as evidence rather than decoration
- Core work:
  - walk through one attribution graph
  - explain three high-contribution paths
  - write one ambiguity paragraph
- Outputs:
  - one graph walkthrough
  - one path-by-path explanation
  - one ambiguity note
- Gate: you can state what the graph does not prove

## Week 9: Tracing tooling

- Modules: `M07`
- Goal: understand that tools shape the questions a researcher can ask
- Core work:
  - draw the workflow
  - analyze one tooling bottleneck
  - write one tooling-needs note
- Outputs:
  - one workflow diagram
  - one bottleneck analysis
  - one tooling-needs note
- Gate: you can explain how the tooling layer constrains or enables a line of research

## Week 10: Character and self-modeling

- Modules: `M08`, `M09`, `M10`
- Goal: connect representational analysis to control questions
- Core work:
  - compare persona vectors, introspection signals, and the assistant axis
  - write one critique that separates evidence from interpretation
  - propose one follow-up experiment
- Outputs:
  - one comparative critique
  - one follow-up experiment idea
  - one short memo
- Gate: you do not confuse self-report with direct evidence of internal state

## Week 11: Capstone design

- Modules: `M03`, `M05`, `M06`, `M08`
- Goal: turn understanding into a credible research proposal
- Core work:
  - pick one question that can be extended from the course
  - define the hypothesis, baseline, budget, and stop condition
  - list the likely failure points
- Outputs:
  - one capstone proposal
  - one resource budget
  - one risk list
- Gate: your proposal is small enough to run in two weeks and specific enough to fail

## Week 12: Final research report

- Modules: `M08`, `M09`, `M10`
- Goal: report like an early-stage researcher, not like someone repeating course notes
- Core work:
  - write the final memo
  - prepare a 10-minute spoken presentation
  - write a next-step plan
- Outputs:
  - one final memo
  - one next-step plan
  - one short presentation
- Gate: your report makes a decision, states the evidence, names the risks, and asks for one concrete next action
